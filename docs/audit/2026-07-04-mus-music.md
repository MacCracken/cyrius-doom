# MUS music subsystem — format + security research (P(-1))

**Date**: 2026-07-04 · **For**: v0.31.4 music wiring · **Sources**: [DoomWiki MUS](https://doomwiki.org/wiki/MUS),
[ModdingWiki MUS_Format](https://moddingwiki.shikadi.net/wiki/MUS_Format),
[Just Solve — Doom MUS](http://fileformats.archiveteam.org/wiki/Doom_MUS), Black Book (DMX/sound chapter).
Format verified by decoding `D_E1M1` from the shareware IWAD.

## What music the IWAD ships

`D_E1M1`–`D_E1M9`, `D_INTER`, `D_INTRO`, `D_INTROA`, `D_VICTOR` — all **MUS** lumps (magic `MUS\x1a`).
`GENMIDI` (11908 B) is the OPL2 FM instrument bank (deferred — see Fidelity below).

## MUS format (Paul Radek's DMX)

- **Header** (16 B): `MUS\x1a`(4) · scoreLen u16 · scoreStart u16 · channels u16 · secChannels u16 ·
  instrCount u16 · dummy u16. Then `instrCount` × u16 instrument (GM patch) numbers. Score begins at `scoreStart`.
- **Size ≤ 64 KB** (all 16-bit fields). DOOM1's biggest is `D_E1M8` at 59,535 B.
- **9 usable channels** (0–8 melodic, plus a percussion channel; MUS uses channel index 15 for percussion in
  the byte, remapped to MIDI 9). `D_E1M1` uses channels 0–2 + 15.
- **Score events** — each event byte = `[last:1 | type:3 | channel:4]`, followed by type-specific operands:
  | type | meaning | operands |
  |---|---|---|
  | 0 | release note | 1 B: note (0–127) |
  | 1 | play note | 1 B: `[vol:1 | note:7]`; if vol bit set, +1 B volume (0–127) |
  | 2 | pitch bend | 1 B (0–255, 64 = centre) |
  | 3 | system event | 1 B controller (10–14: all-notes-off, etc.) |
  | 4 | change controller | 2 B: controller, value. ctl 0 = instrument, 3 = volume, 4 = pan |
  | 5 | (measure/unused) | 0 B |
  | 6 | **score end** | 0 B |
  | 7 | (unused) | 0 B |
  - If `last` is set, a **variable-length delay** follows (ticks to wait before the next event group):
    read bytes while the high bit is set, `delay = delay*128 + (b & 0x7f)`.
- **Tick rate: 140 Hz.** At the engine's 35 Hz game tick, 4 MUS ticks elapse per tick; the mixer runs the
  source stage at 11025 Hz, so the sequencer advances 140/11025 MUS ticks per source sample (sample-accurate).

## Security — malicious/corrupt MUS (the untrusted-WAD surface)

No public CVEs specific to MUS parsing, but the parse is attacker-controlled WAD data and must be bounded
exactly like the patch/WAD decoders. Hardening the implementation MUST enforce:

1. **Header size** — reject a lump < 16 B before reading the header; the instrument list
   (`16 + instrCount*2`) must fit the lump.
2. **scoreStart / scoreLen** — clamp the score window to `[min(scoreStart, lumpSize), lumpSize]`; never trust
   `scoreLen` to extend past the lump.
3. **Every event operand read** (note, volume, controller, value, delay byte) — bounds-check `pos < end`
   before each `load8`; stop the sequencer at end-of-score (treat as score-end / loop).
4. **Varlen delay** — cap the byte count (a run of high-bit bytes could otherwise scan the whole lump); bound
   both by `pos < end` and a small iteration guard.
5. **Channel / note / voice indices** — mask channel to 0–15, clamp note 0–127 before indexing the note→freq
   table and the per-channel volume array; cap the active-voice pool (fixed size, LRU-steal like the SFX mixer).
6. **No writes derived from MUS data** — synthesis writes only into the fixed-size mix accumulator; MUS values
   scale amplitude/phase, never index a write target.
7. **Event-processing guard** — a per-tick `while` reading events must have an iteration cap so a delay-free
   run (all `last=0`) can't spin forever within one tick.

A `fuzz_mus` harness (mutated MUS bytes → parse + sequence N ticks) will exercise this, mirroring `fuzz_wad`.

## Fidelity note (deferred)

Real DOOM used **OPL2 FM synthesis** driven by `GENMIDI` (2-operator patches per GM instrument). Faithful
timbre needs an OPL2 emulator + GENMIDI parsing — a large module deferred to a follow-up. v0.31.4 wires in a
**MUS sequencer + a simple polyphonic wavetable synth** (sine + envelope) mixed into the existing PCM output:
it plays the correct notes/timing/melody through the same ALSA (Linux) / `sys_snd_*` (agnos) path as the SFX.
Percussion (channel 15) is skipped in v1 (no drum kit) — tracked as a follow-up alongside GENMIDI/OPL.
