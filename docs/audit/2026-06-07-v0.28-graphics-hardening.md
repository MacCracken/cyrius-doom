# Graphics Review / Hardening / Performance Audit ŌĆö 2026-06-07 (v0.28.0)

**Auditor**: Claude (Opus 4.8, 1M) + multi-agent workflow (51 subagents)
**Version**: v0.28.0 (from v0.27.5, 592,456 B, cycc 6.0.83)
**Scope**: the full render path ŌĆö `render.cyr`, `framebuf.cyr`,
`texture.cyr`, `sprite.cyr`, `status.cyr`, `menu.cyr`, `automap.cyr`
+ the fixed-point math feeding them (`fixed.cyr`, `tables.cyr`).
**Threat model**: malicious / malformed WAD (local only ŌĆö no network,
no savegames, no DEHACKED). Today only the trusted shareware IWAD is
loaded; most findings are latent until PWAD support lands.

## Methodology

A deterministic workflow fanned out **17 finders** across four
dimensions ŌĆö correctness review, DOOM *Black Book* parity,
security/hardening, and O4-independent performance ŌĆö then:

1. **Find** ŌĆö each finder read its assigned subsystem and returned
   evidence-backed findings (file:line + quoted code).
2. **Triage** ŌĆö 67 raw findings de-duplicated to **27 canonical**.
3. **Verify** ŌĆö every finding adversarially verified; critical/high
   findings got a second *refuting* lens. **20 survived** as real.
4. **Synthesize** ŌĆö prioritized into fix-now / defer / reject with a
   roadmap reshuffle.

Each shipped fix was gated: build + 37/37 WAD-free + 73/73 full
tests + `fuzz_wad`/`fuzz_fixed` + **per-frame PPM diff** (E1M1/E1M3/
E1M5 game + automap + intermission, title, menu, skill) against the
0.27.5 baseline + same-toolchain before/after bench.
